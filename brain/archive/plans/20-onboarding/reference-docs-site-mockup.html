Pipeline Orchestration

Noodle pipelines are deterministic sequences of agentic behaviors. Unlike standard chains, a Noodle pipeline enforces a strict four-stage lifecycle for every interaction: Execute, Quality, Reflect, and Meditate. This ensures high-reliability outputs in autonomous multi-agent systems.

ℹ
Steering Agents typically intervene during the Reflect stage, modifying the context before the next execution cycle begins.

The Four Stages

Every Order processed by the Manager goes through these discrete phases. The architecture prevents race conditions by locking the context state during the mutation phases (Execute, Meditate).

1. Execute

The primary agent attempts to fulfill the prompt. Tools are called, and raw output is generated. No validation occurs here.

2. Quality (QA)

A separate, specialized observer agent critiques the output against the defined schema. If the confidence score is below threshold, the pipeline may loop back or escalate.

pipeline: name: "data-extraction-v1" stages: - type: execute agent: "extractor-01" timeout: 30s - type: quality rules: - "must_be_json" - "confidence > 0.9" - type: reflect strategy: "cumulative_memory"

Manager Orchestration

The Manager is the singleton process responsible for routing Orders to available agents. It maintains the global state of the conversation and handles the "handshake" between different pipeline stages.

When an agent enters the Meditate phase, it is essentially compressing its context window to save tokens for future turns.

from noodle.core import Manager, Order mgr = Manager(mode="strict") # Define an order with specific steering parameters order = Order( prompt="Analyze market trends", steering={ "bias": "conservative", "format": "markdown_table" } ) result = mgr.dispatch(order) print(result.trace_id)
