# Specify the Verification Boundary

When delegating work, state exactly how the delegate can verify their output and what's off-limits. If binaries can't be executed, if certain tools aren't available, if the environment has restrictions — say so upfront.

Don't let delegates discover limitations by trial and error. Every failed verification attempt is a wasted turn and a potential source of confusion or incorrect workarounds.

This applies [[principles/prove-it-works]] at delegation boundaries — the delegate must know how to verify, not just that they should.

Applies to: any delegation where the delegate's environment differs from the delegator's expectations.
