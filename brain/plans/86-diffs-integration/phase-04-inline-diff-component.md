# Phase 4 — Inline Diff Component in Activity Feed

## Goal

Build a collapsible inline diff component that renders in the session activity feed for Edit/Write events. Collapsed by default — shows filename + `+N/-M` stat badge. Click to expand fetches full diff content via the REST endpoint (phase 3) and renders a syntax-highlighted unified diff. Must work in both ungrouped `MessageRow` and grouped `ToolGroup` contexts.

## Changes

**`ui/src/components/InlineDiff.tsx`** (new) — collapsible diff component:
- **Collapsed state:** filename (last path segment + parent dir for context), colored `+N/-M` badge (green additions, red deletions), chevron indicator
- **Expanded state:** fetches diff content via `GET /api/sessions/{id}/events/{index}/diff` using TanStack Query (lazy — only on expand). Renders `MultiFileDiff` from `@pierre/diffs/react` using the `DiffTheme` wrapper from phase 1. For Edit events, passes `oldFile`/`newFile` constructed from response. For Write events, shows as new-file diff. For Codex `apply_patch`, passes the patch to `PatchDiff`.
- **Animation:** simple height transition on expand/collapse

**`ui/src/client/hooks.ts`** (or new file) — add `useDiffContent(sessionId, eventIndex)` hook wrapping TanStack Query. `enabled: false` by default, triggered when user expands. Caches result so re-expand doesn't refetch.

**`ui/src/components/MessageRow.tsx`** — for Edit-labeled tool events that have `diff_meta`, render `InlineDiff` instead of the plain one-liner summary.

**`ui/src/components/ToolGroup.tsx`** — when a group contains Edit events with `diff_meta`, render each child as `InlineDiff` (collapsed) instead of plain `MessageRow`. The group summary still shows "Edit (x5)" with the existing collapse/expand, but expanded children show individual `InlineDiff` components rather than plain summaries.

**`ui/src/client/types.ts`** (generated by tygo) — regenerate to include `DiffMeta` on `EventLine` and `DiffContent` response type.

## Data Structures

- `DiffMeta` — TypeScript type (generated): `{ file_path: string; additions: number; deletions: number }`
- `DiffContent` — TypeScript type (generated): `{ type: "edit" | "write" | "patch"; index: number; file_path: string; old_string?: string; new_string?: string; content?: string; patch?: string }`
- `InlineDiffProps` — `{ sessionId: string; eventIndex: number; meta: DiffMeta; defaultExpanded?: boolean }`

## Verification

### Static
- `pnpm build` passes
- `pnpm check` passes
- Component renders without errors for events with and without `diff_meta`

### Runtime
- Open a session with Edit events in the feed
- **Ungrouped edit:** collapsed state shows filename + stat badge. Click to expand — loading state, then unified diff renders with syntax highlighting. Click again to collapse.
- **Grouped edits ("Edit (x5)"):** expand the group — each child shows as an `InlineDiff` (collapsed). Click individual children to expand their diffs.
- Network tab: diff content only fetched on first expand. Re-expand uses cache.
- Events without `diff_meta` (Read, Bash, etc.) render unchanged
- Scroll performance: expanding a diff mid-feed doesn't break virtualization
